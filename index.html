Project Analysis & Documentation

1. Dataset & Scope

This dashboard visualizes Use Case #19 from the Financial Services domain. The system is built on the Credit Card Transaction Dataset containing 8,580,255 rows and 44 features. The goal is to process high-volume financial data in real-time to detect anomalies.

2. Architecture (Phase 3)

The pipeline uses Apache Kafka to ingest raw transaction streams. Apache Spark Streaming consumes these messages for ETL (Extract, Transform, Load) processes. The data is then passed to the inference engine or stored in S3/HDFS for batch training.

3. Machine Learning (Phase 4)

We utilized XGBoost because it is highly effective for tabular data and handles the extreme class imbalance (Fraud is less than 1%) using `scale_pos_weight`. The primary evaluation metric selected is PR-AUC (Precision-Recall Area Under Curve).

4. Feature Engineering Strategy

The live simulation demonstrates our key features:
• Geo-Distance: Calculated using the Haversine formula between User and Merchant coordinates.
• Log Amount: Transaction amounts are log-transformed to reduce outlier impact.
• Velocity: Tracking the number of transactions in the last 24 hours.
